# A002 員工訂便當

## 功能說明
員工登入後可瀏覽下週工作日的便當選項，並於截止時間前完成訂餐。

## 使用者角色
- 員工
- 系統管理員

## 功能需求

### 員工
- 瀏覽下週工作日可訂的便當選項
- 選擇指定工作日的便當
- 更新已訂的便當（截止前可更換，以最新一筆為準）
- 取消已訂的便當
- 查看自己的訂餐記錄

### 系統管理員
- 新增供應商
- 依據日期建立菜單（需指定供應商）
- 設定菜單有效期間（起始日～結束日）
- 查詢菜單清單（含歷史菜單）
- 編輯菜單資料
- 取消指定員工的訂餐

## 費用規則
- 便當費用由公司全額負擔，員工不需支付任何費用
- 員工瀏覽便當選項時，不顯示價格資訊
- 價格僅供管理員與後台使用，員工端 API 回傳資料不含價格欄位

## 訂餐規則
- 同一員工同一日只保留一筆有效訂單，重複下單時以最新一筆為準（覆蓋舊訂單）

## 截止規則

### 員工
- 訂餐截止時間為**本週星期五 12:00**（新增 / 修改）
- 員工**取消**訂餐的截止時間為**該訂餐日前一日 16:30**
- 員工訂的是**截止日（星期五）後至下下週五前**，菜單設定為有效的日期之便當
- 可訂日期以管理員設定的菜單有效期間為準，不限定週一至週五（週六、週日若有設定菜單亦可訂）
- 例如：今天是星期六 3/22，截止日為下週五 3/27，畫面顯示 3/28 起有效菜單的便當選項
- 截止後不可新增或修改訂餐；各日訂餐超過前一日 16:30 後不可取消

### 系統管理員
- 管理員取消指定員工訂餐的截止時間為該訂餐日前一日的 **16:30**（A003 批次排程執行前 30 分鐘）
- 超過截止時間後，管理員不可再取消該日訂餐

## API 路徑

| 方法 | 路徑 | 說明 |
|---|---|---|
| GET | /api/orders/menu | 取得下週便當選項 |
| POST | /api/orders | 新增訂餐（若同日已有訂單則覆蓋） |
| PATCH | /api/orders/{id} | 更新已訂的便當（截止前） |
| DELETE | /api/orders/{id} | 取消訂餐 |
| GET | /api/orders/me | 查詢個人訂餐記錄 |
| GET | /api/menus | 管理員查詢菜單清單（可帶 include_history 參數） |
| POST | /api/menus | 管理員建立菜單 |
| PATCH | /api/menus/{id} | 管理員編輯菜單 |
| POST | /api/suppliers | 管理員新增供應商 |
| DELETE | /api/admin/orders/{id} | 管理員取消指定員工訂餐 |

## 流程說明

### 員工訂餐流程
1. 員工登入後進入訂餐頁面
2. 系統檢查目前是否在截止時間（星期五 12:00）前
3. 已截止則顯示截止提示，僅可查看記錄
4. 未截止則顯示下週工作日的便當選項
5. 員工選擇日期與便當
6. 系統檢查該日是否已有訂單：若有則覆蓋舊訂單，若無則新增
7. 系統儲存訂餐記錄

### 員工更新訂餐流程
1. 員工於訂餐頁面選擇已訂的便當並更換
2. 系統檢查是否在截止時間前
3. 已截止則顯示提示，無法更新
4. 未截止則更新訂餐記錄為新選擇的便當

### 員工取消訂餐流程（前後端皆須驗證）
1. 前端載入訂餐記錄時，依各筆訂單的訂餐日判斷是否已過前一日 16:30
   - 已超過截止時間：**隱藏**取消按鈕（不顯示）
   - 未超過截止時間：顯示取消按鈕
2. 員工點擊取消按鈕
3. 前端再次檢查當前時間是否仍在截止時間前（防止頁面停留過久）
4. 前端送出 DELETE /api/orders/{id}
5. 後端驗證是否在該訂餐日前一日 16:30 前
   - 已超過：回傳 403，前端顯示錯誤提示
   - 未超過：執行刪除，回傳成功

### 管理員菜單管理流程
1. 管理員進入菜單管理頁面
2. 查詢菜單清單：帶 `include_history=true` 可包含已過期的歷史菜單
3. 選取菜單後可編輯便當名稱、類型、說明、價格、有效期間

### 管理員取消員工訂餐流程
1. 管理員選擇欲取消的員工訂單
2. 系統檢查是否在該訂餐日前一日的 16:30（批次排程執行前 30 分鐘）前
3. 已超過截止時間則顯示提示，不允許取消
4. 未超過截止時間則執行取消，系統刪除指定訂餐記錄

## 資料表

### suppliers（供應商資料表）
- id（PK）
- name（供應商名稱）
- email（通知信箱）
- phone（聯繫電話）
- contact_person（負責人）
- business_registration_no（營業登記編號）
- is_active（是否啟用）
- created_at（建立時間）

### menus（菜單資料表）
- id（PK）
- supplier_id（FK → suppliers.id）
- name（便當名稱）
- category（類型：肉類 / 海鮮 / 素食）
- description（說明）
- price（價格）
- valid_from（有效起始日）
- valid_to（有效結束日）
- created_by（FK → employees.id，建立者）
- created_at（建立時間）
- updated_at（更新時間）

### orders（訂餐記錄資料表）
- id（PK）
- employee_id（FK → employees.id，訂餐員工）
- menu_id（FK → menus.id）
- order_date（訂餐日期）
- created_by（FK → employees.id，實際操作者，管理員代訂時為管理員，否則同 employee_id）
- created_at（建立時間）
