# A002 員工訂便當

## 功能說明
員工登入後可瀏覽下週工作日的便當選項，並於截止時間前完成訂餐。

## 使用者角色
- 員工
- 系統管理員

## 功能需求

### 員工
- 瀏覽下週工作日可訂的便當選項
- 選擇指定工作日的便當
- 取消已訂的便當
- 查看自己的訂餐記錄

### 系統管理員
- 新增供應商
- 依據日期建立菜單（需指定供應商）
- 設定菜單有效期間（起始日～結束日）
- 取消指定員工的訂餐

## 截止規則
- 訂餐截止時間為當週星期五 12:00 前
- 截止後不可新增或取消訂餐

## API 路徑

| 方法 | 路徑 | 說明 |
|---|---|---|
| GET | /api/orders/menu | 取得下週便當選項 |
| POST | /api/orders | 新增訂餐 |
| DELETE | /api/orders/{id} | 取消訂餐 |
| GET | /api/orders/me | 查詢個人訂餐記錄 |
| POST | /api/menus | 管理員建立菜單 |
| POST | /api/suppliers | 管理員新增供應商 |
| DELETE | /api/admin/orders/{id} | 管理員取消指定員工訂餐 |

## 流程說明
1. 員工登入後進入訂餐頁面
2. 系統檢查目前是否在截止時間（星期五 12:00）前
3. 已截止則顯示截止提示，僅可查看記錄
4. 未截止則顯示下週工作日的便當選項
5. 員工選擇日期與便當
6. 系統儲存訂餐記錄
7. 員工可取消已訂的便當（截止前）

## 資料表

### suppliers（供應商資料表）
- id（PK）
- name（供應商名稱）
- email（通知信箱）
- phone（聯繫電話）
- contact_person（負責人）
- business_registration_no（營業登記編號）
- is_active（是否啟用）
- created_at（建立時間）

### menus（菜單資料表）
- id（PK）
- supplier_id（FK → suppliers.id）
- name（便當名稱）
- category（類型：肉類 / 海鮮 / 素食）
- description（說明）
- price（價格）
- valid_from（有效起始日）
- valid_to（有效結束日）
- created_by（FK → employees.id，建立者）
- created_at（建立時間）

### orders（訂餐記錄資料表）
- id（PK）
- employee_id（FK → employees.id）
- menu_id（FK → menus.id）
- order_date（訂餐日期）
- created_at（建立時間）
