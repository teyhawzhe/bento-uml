# A013 員工訂餐報表查詢與 PDF 下載

## 功能說明
管理員可於獨立報表畫面，依日期範圍查詢員工訂餐資料，支援畫面預覽（view）與下載 PDF 兩種方式。

## 使用者角色
- 系統管理員

## 功能需求

### 系統管理員
- 選擇日期起訖範圍查詢員工訂餐資料
- 於畫面上預覽查詢結果
- 選擇排序方式（預設依日期排序，可切換）
- 下載 PDF 格式的訂餐報表

## 報表欄位
每筆資料顯示以下欄位：
- 日期（order_date）
- 部門（department_name）
- 員工姓名（employee_name）
- 便當名稱（menu_name）
- 廠商（supplier_name）

## 排序方式
| 排序選項 | 說明 |
|---|---|
| 日期（預設） | 依 order_date 升冪排序 |
| 部門 | 依部門名稱字母排序 |
| 員工姓名 | 依員工姓名字母排序 |
| 廠商 | 依廠商名稱字母排序 |

## API 路徑

| 方法 | 路徑 | 說明 |
|---|---|---|
| GET | /api/admin/reports/orders | 查詢員工訂餐報表（JSON，可帶 date_from、date_to、sort_by 參數） |
| GET | /api/admin/reports/orders/pdf | 下載員工訂餐報表 PDF（可帶 date_from、date_to、sort_by 參數） |

## 流程說明

### 查詢與預覽流程
1. 管理員進入報表查詢頁面
2. 選擇日期起訖範圍（date_from、date_to）
3. 選擇排序方式（預設：日期）
4. 系統呼叫 `GET /api/admin/reports/orders` 回傳報表資料
5. 後端查詢 orders JOIN employees JOIN departments JOIN menus JOIN suppliers
6. 畫面顯示查詢結果（日期、部門、員工姓名、便當名稱、廠商）

### 下載 PDF 流程
1. 管理員於報表頁面點擊「下載 PDF」
2. 系統以目前選擇的日期範圍與排序方式呼叫 `GET /api/admin/reports/orders/pdf`
3. 後端產生 PDF 並回傳二進位檔案
4. 瀏覽器觸發下載

## 資料表
無新增資料表，資料來源：
- orders（訂餐記錄）
- employees（員工姓名）JOIN departments（部門名稱）
- menus（便當名稱）JOIN suppliers（廠商名稱）
