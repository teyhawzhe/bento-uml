# A009 前端介面 TAB 分頁與訊息提示規範

## 功能說明

前端介面的每個功能模組以 TAB 頁籤區分，避免所有功能擠在同一頁面。
系統的所有提示訊息（成功、失敗、警告、確認）均以 MessageBox 彈窗方式顯示。

## 使用者角色

- 員工
- 管理員

## TAB 分頁規範

### 員工 TAB 頁籤

| 頁籤名稱 | 對應功能 |
|---|---|
| 訂便當 | 員工訂餐（A002） |
| 我的訂單 | 查詢個人訂單 |

### 管理員 TAB 頁籤

| 頁籤名稱 | 對應功能 |
|---|---|
| 訂單管理 | 代替員工新增訂餐與查詢訂單（A006） |
| 供應商管理 | 管理供應商（A007） |
| 報表設定 | 月結報表收件信箱設定（A008） |
| 系統設定 | 系統錯誤通知信箱設定（A004） |

## MessageBox 提示規範

| 情境 | 類型 | 說明 |
|---|---|---|
| 操作成功 | 成功（Success） | 顯示操作成功訊息，使用者點擊確認後關閉 |
| 操作失敗 | 錯誤（Error） | 顯示錯誤原因，使用者點擊確認後關閉 |
| 表單驗證失敗 | 警告（Warning） | 顯示欄位驗證錯誤訊息，使用者點擊確認後關閉 |
| 刪除確認 | 確認（Confirm） | 顯示確認刪除提示，使用者選擇確認或取消 |

## 流程說明

1. 使用者登入後，系統依角色**直接導向對應的 TAB 主頁**，不顯示員工入口／管理員入口選擇頁
2. 使用者點擊 TAB 切換功能頁面，不重新整理整頁
3. 各功能操作完成後，系統以 MessageBox 顯示結果
4. 使用者點擊 MessageBox 確認按鈕後關閉彈窗
5. 若為刪除操作，MessageBox 需有確認與取消兩個按鈕

## 入口頁移除說明

- 登入成功後不再呈現「員工入口」與「管理員入口」選擇頁面
- 系統根據 JWT 中的角色資訊自動判斷並跳轉至對應主頁
  - 角色為員工 → 直接進入員工 TAB 主頁（預設顯示「訂便當」頁籤）
  - 角色為管理員 → 直接進入管理員 TAB 主頁（預設顯示「訂單管理」頁籤）
