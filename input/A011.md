# A011 管理員以部門篩選員工清單

## 功能說明
系統管理員在員工清單頁面，可選擇部門作為查詢條件，篩選出該部門的員工資料。

## 使用者角色
- 系統管理員

## 功能需求

### 系統管理員
- 在員工清單頁面，透過下拉選單選擇部門進行篩選
- 選擇「全部」時回傳所有員工
- 選擇特定部門時，僅回傳該部門的員工

## API 路徑

| 方法 | 路徑 | 說明 |
|---|---|---|
| GET | /api/admin/employees?department_id={id} | 依部門篩選員工清單（不帶參數時回傳全部） |
| GET | /api/admin/departments | 取得部門清單（供下拉選單使用） |

## 流程說明

### 依部門篩選員工流程
1. 管理員進入員工管理頁面
2. 系統呼叫 `GET /api/admin/departments` 取得部門清單，渲染下拉選單（含「全部」選項）
3. 管理員選擇部門
4. 前端帶入 `department_id` 參數呼叫 `GET /api/admin/employees?department_id={id}`
5. 後端依 `department_id` 篩選員工資料並回傳
6. 頁面顯示篩選結果

### 查詢全部員工流程
1. 管理員選擇「全部」
2. 前端不帶 `department_id` 呼叫 `GET /api/admin/employees`
3. 後端回傳所有員工清單
4. 頁面顯示全部員工

## 資料表
（沿用現有資料表，無新增欄位）

### employees（員工資料表）
- id（PK）
- username（帳號）
- name（姓名）
- email
- is_admin
- is_active
- department_id（FK → departments.id）

### departments（部門資料表）
- id（PK）
- name（部門名稱）
