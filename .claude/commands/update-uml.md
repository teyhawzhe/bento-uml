# update-uml

當需求有新增或修改時，執行此指令以重新產出對應需求的所有 UML 圖。

## 使用方式

```
/update-uml A001
```

可同時指定多個需求：

```
/update-uml A001 A002
```

## 執行步驟

1. 讀取指定需求編號對應的 `input/{編號}.md`，理解最新需求內容
2. 讀取 `output/{編號}/` 下現有的 4 個 `.puml` 檔案，了解目前內容
3. 對照需求文件，逐一更新下列 4 個檔案：

| 檔案 | 職責 |
|---|---|
| `output/{編號}/{編號}-usecase.puml` | 列出所有角色與可執行的功能 |
| `output/{編號}/{編號}-activity.puml` | 描述主要流程的步驟、判斷與分支 |
| `output/{編號}/{編號}-sequence.puml` | 描述使用者、系統、資料庫之間的互動順序 |
| `output/{編號}/{編號}-er.puml` | 描述資料表的欄位與關聯（含 PK/FK） |

4. 更新原則：
   - 只修改與需求異動相關的部分，不異動無關內容
   - 所有標籤、說明文字一律使用繁體中文
   - API 路徑依需求文件定義為準，不自行命名
   - 不補充需求文件沒有提到的功能

5. 同步根目錄統整檔案：
   - 執行 `/sync-usecase` 邏輯，重新產出根目錄的 `usecase.puml`
   - 執行 `/sync-openapi` 邏輯，重新產出根目錄的 `openapi.yaml`

6. 更新完成後列出每個檔案的異動摘要

## 注意

- 若需求文件（input/）尚未更新，請先更新需求文件再執行此指令
- usecase.puml 與 openapi.yaml 會在每次執行後自動同步，無需另行執行
